❓ Frequently Asked Questions (Viva / Q&A)

🎯 Q1: How do you calculate the accuracy of the model?
Accuracy = (Number of Correct Predictions) / (Total Number of Predictions) × 100

We use categorical accuracy since this is a multi-class classification problem. For each test image, the model outputs a probability distribution over 7 classes. The predicted class is the one with the highest probability (argmax). If this matches the ground truth label, it's counted as correct.

Formula: Accuracy = TP + TN / (TP + TN + FP + FN) for each class, then averaged.
We also compute weighted average to account for class imbalance in HAM10000 (nv has 6,705 images vs df with 115).
📊 Q2: What is the expected accuracy of your model?
Based on our custom CNN architecture (4 conv blocks, 1.4M parameters) and literature comparisons on HAM10000:
• Training Accuracy: ~90–95%
• Validation Accuracy: ~82–88%
• Test Accuracy: ~80–85%

Note: Using transfer learning (e.g., EfficientNet, ResNet-50) can boost test accuracy to 90%+. The class imbalance in HAM10000 (67% Melanocytic Nevi) requires data augmentation and class weighting for fair evaluation.
🧠 Q3: Why did you choose CNN over other machine learning algorithms?
CNNs are the gold standard for image classification because:
• Automatic feature extraction — CNNs learn relevant features (edges, textures, patterns) directly from images, unlike traditional ML which requires manual feature engineering
• Spatial hierarchy — Convolutional layers capture local patterns, and deeper layers combine them into higher-level features
• Proven in dermatology — Esteva et al. (2017, Nature) demonstrated CNN performance matching 21 board-certified dermatologists
• Traditional ML (SVM, Random Forest) would require manually extracting features like color histograms, texture descriptors (GLCM), which is less effective.
🔐 Q4: How does the authentication system work?
We use JWT (JSON Web Token) based stateless authentication:
1. Signup: User provides name, email, password → password is hashed using bcrypt (one-way hash with salt) → stored in MongoDB
2. Login: Email + password submitted → bcrypt verifies hash → if valid, server generates a JWT token containing user email, role, and expiration
3. Protected Routes: Client sends JWT in Authorization header → server decodes and verifies the token → grants access
4. Security: Passwords are never stored in plaintext. JWT tokens expire after a configurable time (default: 24 hours)
📦 Q5: What is the HAM10000 dataset and why did you choose it?
HAM10000 (Human Against Machine with 10,000 training images) was published by Tschandl et al. in 2018 in Scientific Data (Nature). It contains:
• 10,015 dermatoscopic images
• 7 clinically important diagnostic categories
• Verified by histopathology in over 50% of cases
• It's the standard benchmark for skin disease classification research, used in the annual ISIC challenge at MICCAI conference.
⚙️ Q6: What is Batch Normalization and why do you use it?
Batch Normalization (Ioffe & Szegedy, 2015) normalizes the output of each layer by adjusting and scaling the activations:
• Purpose: Reduces internal covariate shift — stabilizes the distribution of inputs to each layer during training
• Benefits: Allows higher learning rates, acts as regularization, reduces dependence on careful weight initialization
• How it works: For each mini-batch, it computes mean and variance, then normalizes: x̂ = (x - μ) / √(σ² + ε), followed by learnable scale γ and shift β
🛑 Q7: What is Dropout and what rate did you use?
Dropout (Srivastava et al., 2014) is a regularization technique that randomly sets a fraction of neuron outputs to zero during training:
• Prevents overfitting by ensuring the network doesn't rely on any single neuron
• Our rates: 0.25 in early conv blocks, 0.30 in deeper conv blocks, and 0.50 in fully connected layers
• At inference time, all neurons are active but their outputs are scaled by (1 - dropout_rate)
• Higher dropout in FC layers because they have more parameters and are more prone to overfitting
📐 Q8: What is the role of Global Average Pooling (GAP)?
Global Average Pooling replaces traditional fully connected layers as the bridge between convolutional and classification layers:
• It averages each feature map into a single value — e.g., from 14×14×256 to 1×1×256 (just 256 values)
• Advantages over Flatten: Drastically reduces parameters (prevents overfitting), provides spatial invariance, works as a structural regularizer
• Used in modern architectures like GoogLeNet, ResNet, and EfficientNet
🔄 Q9: Why is Data Augmentation important for this project?
The HAM10000 dataset has severe class imbalance (nv: 6,705 vs df: 115 images). Data augmentation helps by:
• Increasing effective training set size — generating new variations of existing images
• Reducing overfitting — model sees different versions each epoch
• Improving generalization — rotation, flip, zoom simulate real-world image variations
• Addressing class imbalance — minority classes (df, vasc) get more augmented samples, improving recall for rare diseases
🏗️ Q10: Why did you choose FastAPI over Flask?
FastAPI was chosen for several technical advantages:
• Performance: Built on Starlette and Uvicorn (ASGI) — 3-10x faster than Flask (WSGI)
• Async support: Native async/await for handling concurrent requests (important for ML inference)
• Auto-documentation: Generates Swagger UI at /docs automatically
• Type validation: Uses Pydantic for automatic request/response validation
• Modern Python: Type hints, dependency injection, middleware support out of the box
