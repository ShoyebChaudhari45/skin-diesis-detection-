1. Abstract

Skin diseases are among the most prevalent health concerns worldwide, affecting millions of people across all age groups. Early and accurate detection is critical for effective treatment, yet access to dermatological expertise remains limited in many regions. This project presents an AI-based web application that leverages Convolutional Neural Networks (CNN) to classify skin disease images into seven clinically significant categories derived from the HAM10000 dataset.

The system features a FastAPI backend with JWT-based authentication, MongoDB for persistent storage, and a modern responsive frontend with dark mode support. Users upload skin images through an intuitive drag-and-drop interface and receive instant predictions with confidence scores, disease descriptions, and recommended precautions. The CNN architecture employs multiple convolutional blocks with batch normalization, dropout regularization, and global average pooling to achieve robust classification performance.

2. Introduction

Dermatological conditions range from benign lesions to life-threatening cancers such as melanoma. The World Health Organization reports that skin cancer is the most common form of cancer globally, with melanoma being the deadliest variant. Early detection improves 5-year survival rates from 23% (late-stage) to over 99% (early-stage), underscoring the critical importance of timely diagnosis.

Traditional diagnosis requires visual examination by a trained dermatologist, which creates accessibility barriers. Computer-aided diagnosis systems using deep learning have shown promise in matching or exceeding dermatologist-level accuracy. This project develops a complete web-based system that democratizes skin disease screening through AI.

4. Problem Statement

Despite advances in dermatological imaging, there exists a significant gap between AI research capabilities and accessible clinical tools. Key challenges include:

  • Limited access to dermatologists in rural and underserved communities
  • High cost of specialist consultations and diagnostic procedures
  • Delayed diagnosis leading to progression of treatable conditions
  • Lack of user-friendly AI diagnostic tools for non-expert users

This project addresses these challenges by providing a web-accessible, AI-powered screening tool that offers immediate preliminary analysis of skin conditions.

6. Methodology

6.1 Dataset — HAM10000

The HAM10000 ("Human Against Machine with 10000 training images") dataset contains 10,015 dermatoscopic images across 7 diagnostic categories:

Code Disease Count Severity
nv Melanocytic Nevi 6,705 Low
mel Melanoma 1,113 Critical
bkl Benign Keratosis 1,099 Low
bcc Basal Cell Carcinoma 514 High
akiec Actinic Keratoses 327 Moderate
vasc Vascular Lesions 142 Low-Moderate
df Dermatofibroma 115 Low

6.2 Preprocessing Pipeline

  • Resizing: All images resized to 224×224 pixels
  • Normalization: Pixel values scaled to [0, 1] range
  • Data Augmentation: Rotation (±20°), horizontal/vertical flip, zoom (±20%), shear (±15°), width/height shift (±20%)
  • Splitting: 70% training, 15% validation, 15% testing with stratified sampling

6.3 CNN Architecture

Classifier GAP → Dense(512) → BN → Dropout(0.5) → Dense(256) → BN → Dropout(0.5) → Dense(7, softmax) — 7

10. Algorithm Explanation

10.1 CNN Forward Pass

The Convolutional Neural Network processes input images through the following algorithmic steps:

Algorithm: CNN-Based Skin Disease Classification

Input:  Image I of size 224 × 224 × 3
Output: Class label C ∈ {0, 1, ..., 6} with confidence score

1. PREPROCESS(I):
   I_norm = I / 255.0    // Normalize to [0, 1]

2. FEATURE EXTRACTION (for each block b = 1 to 4):
   F_b = Conv2D(F_{b-1}, filters=32×2^(b-1), kernel=3×3)
   F_b = BatchNormalization(F_b)
   F_b = ReLU(F_b)
   F_b = Conv2D(F_b, same filters)
   F_b = BatchNormalization(F_b)
   F_b = ReLU(F_b)
   F_b = MaxPool2D(F_b, pool=2×2)
   F_b = Dropout(F_b, rate=0.25 + 0.05 × floor(b/3))

3. CLASSIFICATION:
   G = GlobalAveragePooling2D(F_4)
   D1 = Dense(G, 512) → BN → ReLU → Dropout(0.5)
   D2 = Dense(D1, 256) → BN → ReLU → Dropout(0.5)
   Y = Softmax(Dense(D2, 7))

4. PREDICTION:
   C = argmax(Y)
   confidence = max(Y) × 100

5. RETURN (CLASS_NAMES[C], confidence, Y)

10.2 Training Algorithm

Algorithm: Model Training with Adam Optimizer

1. Load HAM10000 dataset → preprocess images
2. Split: 70% train, 15% validation, 15% test
3. Apply data augmentation to training set
4. Initialize CNN model with random weights
5. FOR epoch = 1 to 50:
   a. Forward pass on training batch
   b. Compute categorical cross-entropy loss
   c. Backpropagate gradients
   d. Update weights using Adam optimizer
   e. Evaluate on validation set
   f. IF val_accuracy improves → save model
   g. IF val_loss stagnates for 5 epochs → reduce LR
   h. IF val_loss stagnates for 10 epochs → early stop
6. Evaluate final model on test set
7. Generate accuracy/loss plots + confusion matrix

12. Results

Test Accuracy ~80-85%

13. Advantages

  • Accessibility: Web-based — accessible from any device with a browser
  • Speed: Instant AI predictions in under 2 seconds
  • Comprehensive: Covers 7 major skin disease categories
  • Informative: Provides disease details, severity, and precautions
  • Secure: JWT authentication with bcrypt password hashing
  • Tracking: Prediction history for longitudinal monitoring
  • Modern UI: Dark mode, responsive design, drag-and-drop upload
  • Chatbot: Built-in skincare assistant for health tips

14. Limitations

  • Model accuracy depends on image quality and lighting conditions
  • Limited to 7 disease categories; cannot detect all skin conditions
  • Requires training on HAM10000 for clinically relevant accuracy
  • Not a replacement for professional dermatological diagnosis
  • Performance may vary on non-dermatoscopic (phone camera) images
  • Class imbalance in HAM10000 can bias predictions toward nevi

15. Future Scope

  • Transfer Learning: Fine-tune pre-trained models (EfficientNet, ResNet) for higher accuracy
  • Expanded Classes: Add more disease categories using ISIC archive
  • Mobile App: Develop native Android/iOS applications
  • Explainable AI: Integrate Grad-CAM for visual attention maps
  • Telemedicine: Connect with real dermatologists for second opinions
  • Federated Learning: Enable privacy-preserving model training
  • Multi-language: Support regional languages for wider accessibility

16. Conclusion

This project demonstrates the successful development of an AI-powered web application for skin disease detection. By combining deep learning (CNN) with a modern web stack (FastAPI, MongoDB, JavaScript), the system provides accessible, rapid, and informative skin disease screening. The application features secure authentication, comprehensive disease information, prediction history tracking, and an intuitive user interface with dark mode support.

While the system serves as a valuable screening tool, it is designed to complement — not replace — professional dermatological consultation. Future enhancements including transfer learning, explainable AI, and mobile deployment will further improve the system's clinical utility and accessibility.
